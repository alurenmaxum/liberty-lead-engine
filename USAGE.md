# Liberty Lead Engine — Usage Guide

**Live app:** https://liberty-pearl-six.vercel.app

This guide is for using the dashboard.
It assumes you do **not** know anything about the code or how the app was built.

---

## 1. How to log in

1. Open the live app:
   - https://liberty-pearl-six.vercel.app
2. You will be taken to the sign-in page.
3. Enter the admin login details.
4. After signing in, you will land in the dashboard.

If you are not logged in, the app will send you back to the sign-in page automatically.

---

## 2. What the main dashboard pages do

At the top of the dashboard, you will see navigation links:

- **Pipeline** — your main lead list
- **Simulate** — create test leads and test the qualification flow
- **Bot Control** — manage keyword rules and message templates
- **Nurture** — manage follow-up sequences
- **Analytics** — view metrics, funnel data, trends, and campaign performance

If you are unsure where to start, start with:
1. **Simulate** to create a test lead
2. **Pipeline** to view that lead
3. Click the lead to open the full detail page

---

## 3. How to simulate a lead coming in

The system does not currently use live WhatsApp. Instead, it uses a **simulation flow**.

### Using the dashboard

1. Open **Simulate**.
2. Fill in the test details shown on the page.
3. Submit the form/message.
4. The app will create or update a lead and move the bot conversation forward.
5. You will see the response message generated by the bot.

You can repeat this to move the same lead through the qualification steps.

### What this means in plain English

You are pretending to be a lead sending WhatsApp-style messages.
The system reads the message, updates the lead, and replies as if the bot handled it.

### The intake endpoint

Behind the dashboard, the app uses this endpoint:

- `POST /api/simulate/intake`

You do **not** need to call this manually if you are using the dashboard.
The Simulate page does that for you.

## 4. How to navigate the pipeline / lead list

1. Click **Pipeline**.
2. You will see leads grouped by stage.
3. Each lead card shows the current state of that lead.
4. Use this page to quickly see:
   - new leads
   - leads still qualifying
   - leads ready for booking
   - booked or later-stage leads

### What to look for

Each lead has a stage, for example:
- qualifying
- booking
- booked
- later pipeline stages

The pipeline helps you see where each lead is sitting right now.

### Opening a lead

Click a lead to open the full lead detail page.
That is where you see the conversation, captured data, and any adviser briefing.

---

## 5. How to view a lead's conversation and qualification data

1. Go to **Pipeline**.
2. Click the lead you want.
3. On the lead detail page, you can review:
   - the conversation history
   - the qualification answers already captured
   - the lead’s current stage/status

### Conversation history

You will see the back-and-forth messages for that lead.
This lets you understand:
- what the lead said
- what the bot replied
- where the lead dropped off or progressed

### Qualification data

The page also shows the lead information collected during the flow, such as:
- first name
- age or age band
- interests
- any other qualification fields captured during the chat

This is the main place to understand what the system already knows about a lead.

---

## 6. How to read the adviser briefing

When a lead has progressed far enough, the system prepares a short adviser-style summary.

### Where to find it

Open the lead detail page from **Pipeline**.
If a briefing has been created for that lead, it will appear there.

### What the briefing is for

It gives the adviser a quick summary before speaking to the lead.
That saves time and avoids rereading the full conversation.

### What it usually includes

In plain language, it is meant to answer:
- who the lead is
- what they are interested in
- how warm or serious they seem
- what happened in the conversation so far
- what the adviser should know before contacting them

If no briefing is shown yet, it usually means the lead has not reached the point where one is generated.

## 7. How the bot control panel works

Click **Bot Control**.
This area is for managing how the bot behaves.

There are two main parts:

### A. Keyword rules

This lets you manage compliance-style keyword lists.
These rules are used to block or flag certain kinds of wording.

Use this when you want to:
- add blocked phrases
- add phrases that should be flagged for caution
- tighten or loosen the bot’s message controls

### B. Templates

Templates are reusable message drafts.
They are used by the system for approved outbound messaging.

Typical template workflow:
1. Create a template
2. Review it
3. Approve it
4. Use it in nurture sequences

If a template is not approved, it should not be treated as ready for live use.

### Starting / stopping the bot

There is no separate big on/off switch in the current dashboard.
Instead:
- the simulated intake flow drives the bot
- template approval and keyword rules affect what the system can send
- nurture enrollments can be paused/resumed from the nurture area

So in practical terms, “bot control” here means controlling the bot’s content and guardrails, not flipping a single master power button.

---

## 8. How nurture sequences work

Click **Nurture**.

This area is for follow-up sequences after the first qualification flow.
Think of it as a structured follow-up plan.

### What a sequence is

A nurture sequence is a list of steps the system can use to follow up with a lead over time.

A sequence can include things like:
- sending an approved template
- AI-generated follow-up content
- other simple actions defined in the sequence builder

### What you can do here

- create a new sequence
- edit an existing sequence
- review current sequences
- view active enrollments
- pause or resume enrollments where supported

### In plain English

If the lead is not ready right now, nurture sequences help keep the conversation warm without manually messaging every person one by one.

---

## 9. How to send a manual reply to a lead

On the lead detail page, there is a manual reply box.

Use it when you want to send a human message instead of relying on the bot.

### Steps

1. Open the lead from **Pipeline**.
2. Scroll to the conversation section.
3. Type your message into the reply box.
4. Click **Send**.

The message will be saved in the conversation as a human-sent outbound message.

This is useful when:
- you want to take over manually
- the lead needs a custom answer
- you do not want the automated flow to handle the next step

## 10. How to use Analytics

Click **Analytics**.

This page gives you a higher-level view of what is happening in the system.

### What you will see

- KPI cards
- funnel view
- trend chart
- campaign table
- date range picker
- CSV export link

### What each part means

#### KPI cards
Quick totals like:
- leads created
- booked leads
- closed leads
- ad spend
- average cost per lead
- conversion rate

#### Funnel
Shows how many leads made it from one step to the next.
Useful for spotting where people drop off.

#### Trend chart
Shows daily movement over time.
Useful for seeing whether things are improving or slowing down.

#### Campaign table
Shows performance by source/campaign.
Useful for comparing where leads are coming from.

#### Export CSV
Downloads the visible analytics data into a spreadsheet-friendly file.

## 11. Non-obvious but useful things to know

### The system is currently a POC
This is a working proof of concept, not a final production operating system.
Some parts are simplified on purpose.

### Simulated intake is expected
If you do not see live WhatsApp, that is normal.
Use **Simulate** to test the full flow.

### Analytics may start empty
If there is little or no data yet, the analytics page may show zeros or sparse charts.
That does not necessarily mean something is broken.
It may just mean not enough leads have been processed yet.

### Template approval matters
If you create templates, remember to approve them before expecting them to behave like ready-to-use content.

### Manual reply is available
You do not have to rely entirely on automation.
You can step in on a lead and send a human message.

### If something looks “stuck”
Usually the best check is:
1. Go to **Pipeline**
2. Open the lead
3. Read the conversation history
4. Check the current stage

That usually tells you where the lead is in the flow.

## 12. Recommended first-time walkthrough

If this is your first time using the app, do this:

1. Log in
2. Open **Simulate**
3. Create a test lead
4. Send a few follow-up simulated messages
5. Open **Pipeline**
6. Click the new lead
7. Review the conversation and qualification details
8. Try a manual reply
9. Open **Bot Control** and look at templates/keyword rules
10. Open **Nurture** and inspect the sequences
11. Open **Analytics** and review the dashboard

That will give you the full picture of how the system is meant to be used.

---

## 13. Short version

If you only remember three things:

- **Simulate** creates test leads and moves the bot flow forward
- **Pipeline** is where you manage and open leads
- **Lead detail** is where you see the conversation, captured info, briefing, and manual reply
